# コードレビュー

以下の観点でコードをレビューし、「なぜそうすべきか」も含めて説明する。

## チェック項目

### 1. CLAUDE.md準拠

- セマンティックトークン使用（bg-card, text-foreground等）
- 8pxグリッド準拠
- any型禁止

**なぜ**: セマンティックトークンを使うことで、ダークモード対応やテーマ変更が容易になる。any型は型安全性を破壊し、実行時エラーの原因になる。

### 2. Next.js公式準拠

- Server/Client Componentの適切な使い分け
- next/image使用
- App Router パターン

**なぜ**: Server Componentはバンドルサイズを削減し、初期表示を高速化する。Client Componentは必要最小限にすべき（Googleの推奨）。

### 3. セキュリティ

- XSS対策
- 入力バリデーション
- 認証・認可

**なぜ**: OWASP Top 10に含まれる脆弱性は、ユーザーデータの漏洩や改ざんにつながる。Zodでの入力検証は必須。

### 4. パフォーマンス

- 不要な再レンダリング
- 適切なメモ化
- バンドルサイズ

**なぜ**: 不要な再レンダリングはUXを損なう。React DevToolsのProfilerで確認できる。

### 5. 可読性・保守性

- 関数の責務が単一か
- 命名が適切か
- コメントは必要最小限か

**なぜ**: コードは書く時間より読む時間の方が長い。将来の自分（や他の開発者）が理解しやすいコードを書く。
