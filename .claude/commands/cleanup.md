# 不要コード削除

不要なコード、ファイル、機能を安全に削除する。

## 削除対象の特定

### CLAUDE.mdの削除候補を確認

まず `CLAUDE.md` の「🗑️ 削除候補」セクションを確認する。

### 追加の検出項目

- 未使用のexport
- 参照されていないファイル
- デッドコード
- 古いコメント（TODO、FIXME）

## 削除手順

1. **影響範囲の確認**
   - 削除対象を参照しているファイルを検索
   - テストへの影響を確認

2. **段階的な削除**
   - まずexportを削除
   - 参照エラーがないか確認
   - ファイル全体を削除

3. **検証**
   - `npm run typecheck` - 型エラーがないか
   - `npm run lint` - Lintエラーがないか
   - 画面が正常に動作するか

## なぜ削除するのか（開発者向け説明）

不要なコードを残すデメリット：

- **認知負荷**: 読む人が「これは使われているのか？」と悩む
- **保守コスト**: 変更時に不要なコードも考慮しなければならない
- **バグのリスク**: 使われていないと思ったコードが実は使われていた

GAFAの原則：**YAGNI (You Ain't Gonna Need It)**
「いつか使うかも」で残さない。必要になったらまた作る。

## 出力形式

```markdown
## 削除レポート

### 削除したファイル

- `path/to/file1.ts` - 理由
- `path/to/file2.ts` - 理由

### 削除した機能

- 機能名 - 理由

### 検証結果

- typecheck: ✅ パス
- lint: ✅ パス
```
