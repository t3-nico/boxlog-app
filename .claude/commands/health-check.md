# 技術的健全性チェック

コードベースの技術的な健全性を診断し、問題点と改善提案を出力する。

## チェック項目

### 1. 技術的負債

- TODO/FIXME/HACK/BUGコメント
- @ts-ignore, @ts-nocheck, as any の使用箇所
- 無効化されたテスト（describe.skip, it.skip）
- console.log/console.warn の残存（本番リスク）

### 2. コード品質

- 巨大ファイル（500行以上）
- 重複コード
- 未使用のexport/ファイル
- 型安全性の問題

### 3. セキュリティ

- 認証情報のログ出力
- 入力バリデーションの不足
- XSS/SQLインジェクションのリスク

### 4. パフォーマンス

- 不要な再レンダリング
- N+1クエリの可能性
- バンドルサイズの肥大化

## 実行コマンド

```bash
# 型チェック
npm run typecheck

# Lint
npm run lint

# 依存関係の整合性
npm ls --all | grep -E "UNMET|invalid"

# セキュリティ監査
npm audit
```
