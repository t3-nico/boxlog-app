# コード解説

指定されたファイルやコードについて、「なぜこうなっているか」を解説する。

## 目的

開発者が学びながら開発できるよう、コードの意図・設計思想を説明する。

## 解説の観点

### 1. このコードは何をしているか

- 機能の概要
- データの流れ
- 他のファイルとの関係

### 2. なぜこう書いているか

- GAFAのベストプラクティスとの関連
- 採用している設計パターン
- 他の選択肢との比較

### 3. ツールの役割

- Next.js, Supabase, tRPC, Zustandなどがどう連携しているか
- このファイルがアーキテクチャ全体のどこに位置するか

## 出力形式

```markdown
## [ファイル名] の解説

### 概要

このファイルは〇〇を担当しています。

### データの流れ

1. ユーザーが〇〇する
2. このコードが〇〇を処理
3. 結果が〇〇に渡される

### なぜこう書いているか

- **パターン名**: 〇〇パターンを採用
- **理由**: GAFAでは〇〇が推奨されているため
- **メリット**: 〇〇

### 関連ファイル

- [関連ファイル1](path/to/file1.ts) - 〇〇との連携
- [関連ファイル2](path/to/file2.ts) - 〇〇を参照
```

## ドキュメント化

解説内容は `docs/architecture/` または `docs/learning/` に保存することを提案する。
