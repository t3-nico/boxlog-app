# Entries

タスク・予定の統合エントリ機能。タイムボクシング、繰り返し、充実度記録をサポート。

---

## 概要

| 項目 | 値 |
|------|-----|
| 時間位置 | `upcoming` / `active` / `past`（`getEntryState()` で自動判定） |
| 起源 | `planned`（予定）/ `unplanned`（記録のみ） |
| 時間指定 | `start_time` + `end_time`（ISO 8601、任意） |
| 繰り返し | RRULE ベース（daily / weekly / monthly / yearly） |
| タグ | 多対多（`plan_tags` 経由） |
| リマインダー | 開始N分前（5/10/15/30） |
| 充実度 | `fulfillment_score`（1-5、active/past のみ） |

## 状態管理

### Zustand Stores

| Store | 用途 |
|-------|------|
| `useEntryInspectorStore` | Inspector開閉、entryId、pendingChanges |

## 「Time waits for no one」原則

- `status` フィールドなし — 時間位置から `getEntryState()` で自動判定
- `upcoming`: 予定時間セクション + 繰り返し + リマインダー
- `active`: 予定 + 実績時間 + 充実度 + 繰り返し + リマインダー
- `past`: 予定 + 実績時間 + 充実度 + 繰り返し + リマインダー

---

## 保存フロー

```
パネル開く
  └── entryId セット → usePlan で取得

ユーザー編集
  ├── タイトル/説明 → 即時保存（autoSave）
  └── 日時変更 → pendingChanges にバッファ
      └── 繰り返しエントリの場合 → スコープダイアログ表示

パネル閉じる
  └── saveAndClose()
      └── updatePlan mutation（楽観的更新）
```
