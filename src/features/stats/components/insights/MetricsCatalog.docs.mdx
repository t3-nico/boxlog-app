# Layer 1: メトリクス候補カタログ

entriesテーブルのデータモデルから算出可能な全メトリクスを洗い出し、採否を整理したカタログ。

---

## 採用済み（8メトリクス）

Layer 1で実装するKPIメトリクス。2行×4列グリッドに配置。

### Row 1: 基本指標（量・質・計画・継続）

| メトリクス | 計算式 | DB関数 | 意味 | 採用理由 |
|-----------|--------|--------|------|----------|
| **Total Time** | `SUM(duration)` | `getCumulativeTime`（既存） | 期間内の記録時間合計 | 最も基本的な数字。天気予報における気温。Statsの最上部に出るべき |
| **Avg Fulfillment** | `AVG(fulfillment_score)` | 新規RPC or client計算 | 充実度の全体平均 | Dayoptの独自性そのもの。「時間の質」を測る唯一の指標。Layer 2/3のクロス分析に直結 |
| **Plan Rate** | `planned / total` | `get_plan_rate` | 計画して行動できた割合 | Dayoptの核心。タイムボクシングの実践度を直接測定 |
| **Streak** | 連続アクティブ日数 | `getStreak`（既存） | 継続日数 | モチベーション維持に直結。「明日も途切れさせたくない」。既存DB関数で実装コストほぼゼロ |

### Row 2: 分析指標（精度・ピーク・集中・余白）

| メトリクス | 計算式 | DB関数 | 意味 | 採用理由 |
|-----------|--------|--------|------|----------|
| **Estimation Accuracy** | `avg(\|actual - planned\|)` | `get_estimation_accuracy` | 見積もりと実績のズレ（タグ別） | 自己理解の正確さ。改善可能な具体的アクション |
| **Peak Utilization** | `peak_minutes / peak_available` | `get_energy_map` + client | ピーク時間帯の活用度 | chronotype連携。高エネルギー帯を有効活用しているか |
| **Context Switches** | `tag変化回数 / 日数` | `get_context_switches` | 1日あたりのタスク切替数 | 集中力の分散度。マルチタスクの可視化 |
| **Blank Rate** | `1 - scheduled / available` | `get_blank_rate` | 空白時間の割合 | 余白の意図性。詰め込みすぎ/余裕の判断材料 |

### 選定基準

1. **アクショナブル**: 数値を見てユーザーが行動を変えられる
2. **相互補完**: 8つで「量・質・計画・継続・精度・ピーク・集中・余白」を網羅
3. **シンプル**: 説明なしで直感的に理解できる
4. **グリッド適合**: 8カード = 2行×4列でレスポンシブグリッドが綺麗に収まる（2col / 4col）

## 追加候補（高価値）

追加検討に値するメトリクス。

| メトリクス | 計算式 | データソース | 意味 | 推奨度 | 備考 |
|-----------|--------|-------------|------|--------|------|
| **Completion Rate** | `actual_start IS NOT NULL / planned count` | entries.actual_start_time | 予定のうち実際に着手した割合 | ★★★ | Plan Rateと補完関係（計画→実行のファネル） |
| **Avg Entry Duration** | `AVG(end - start)` | entries.start/end_time | 1エントリの平均時間 | ★★☆ | タイムボクシングの粒度。短すぎ/長すぎの判断 |
| **Top Tag Concentration** | `MAX(tag_time) / total_time` | entries + tags | 上位タグへの時間集中度 | ★★☆ | 多角的 vs 一極集中の可視化 |
| **Overrun Rate** | `actual > planned` の割合 | actual vs planned duration | 予定時間を超過したエントリの比率 | ★★☆ | Estimation Accuracyの補完（方向性あり） |

## 追加候補（中価値）

実装可能だが、優先度は高くないメトリクス。

| メトリクス | 計算式 | 意味 | 推奨度 | 備考 |
|-----------|--------|------|--------|------|
| **Morning/Evening Ratio** | 午前 vs 午後の記録割合 | 朝型/夜型の実態 | ★☆☆ | chronotypeとの答え合わせ |
| **Weekend Activity** | 平日 vs 休日の記録量比 | ワークライフバランス | ★☆☆ | 判断が文化依存。良い/悪いが言えない |
| **Fulfillment Variance** | `STDDEV(fulfillment_score)` | 充実度のばらつき | ★☆☆ | 安定性の指標だが解釈が難しい |
| **Avg Start Delay** | `AVG(actual_start - start_time)` | 予定開始からの平均遅延 | ★☆☆ | actual_start_timeの記録率に依存 |
| **Tag Diversity (Shannon)** | `-Σ(p * log(p))` | タグの多様性指数 | ★☆☆ | 数学的に面白いがユーザーに説明しにくい |
| **Unplanned Rate** | `1 - planRate` | 飛び込みタスクの割合 | — | Plan Rateの逆数。重複 |
| **Planned-Only Fulfillment** | planned entriesのみの充実度平均 | 計画タスクの充実度 | ★☆☆ | Plan Rate × Fulfillmentのクロス |
| **Short Entry Rate** | `duration < 15min` の割合 | 細切れタスクの比率 | ★☆☆ | Context Switchesと部分的に重複 |
| **Long Entry Rate** | `duration > 120min` の割合 | 長時間タスクの比率 | ★☆☆ | タイムボクシング逸脱の検出 |
| **Gap Between Entries** | エントリ間の平均空白時間 | 切り替えの余白 | ★☆☆ | Blank Rateの詳細版 |

## Layer 2向き（Layer 1では不採用）

単体のKPI数値ではなく、パターン分析として表現すべきメトリクス。

| メトリクス | Layer 2での表現 | 理由 |
|-----------|----------------|------|
| 曜日別パターン | DayOfWeek heatmap | 単一数値に落とせない。分布として見る価値 |
| 時間帯別分布 | Hourly distribution chart | 既存チャート（HourlyDistribution）でカバー済み |
| 月間トレンド | Monthly trend line | 既存チャート（MonthlyTrend）でカバー済み |
| 見積もりバイアス推移 | Estimation bias trend line | 週推移ラインとして可視化（Layer 2実装予定） |
| ルーティン遵守率 | Routine adherence list | 繰り返しパターンの検出が前提（Layer 2実装予定） |
| 充実度×時間帯クロス | Fulfillment heatmap | 2Dヒートマップで表現（Layer 2実装予定） |
| 計画外エントリの時間帯集中 | Unplanned pattern chart | 分布パターンとして表現（Layer 2実装予定） |
| タグ間の遷移パターン | Transition matrix | 「AのあとBが多い」は単一数値にならない |
| 曜日×充実度の相関 | Cross-tabulation | 2次元分析はLayer 2向き |

## データモデル参照

Layer 1メトリクスが依存する `entries` テーブルの主要カラム:

| カラム | 型 | メトリクスでの用途 |
|--------|----|--------------------|
| `start_time` / `end_time` | timestamptz | 予定時間（duration計算） |
| `actual_start_time` / `actual_end_time` | timestamptz | 実績時間（精度計算） |
| `origin` | text | `'planned'` / `'unplanned'`（計画率） |
| `fulfillment_score` | int (1-5) | 充実度スコア |
| `tag_ids` | uuid[] | タグ配列（コンテキストスイッチ、タグ別集計） |
| `user_id` | uuid | ユーザー識別 |
