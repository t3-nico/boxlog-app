# Stats

時間記録の統計・分析ビュー。KPIメトリクス、トレンドチャート、パターン分析で自己客観視を支援。

---

## 概要

独立 Feature（`@/features/stats`）として実装。3タブ構成で段階的にデータを深掘りできる。

| タブ | 説明 |
|-----|------|
| **Review（振り返り）** | 期間ベースのKPIメトリクス。日/週/月/年で切り替え |
| **Progress（進捗）** | 累積・長期トレンド。ヒートマップ + 分布チャート |
| **Insights（インサイト）** | AI分析・提案（将来実装） |

### タブと Layer の関係

タブは**データの性質**（期間振り返り / 累積）で分類し、Layer は**計算の複雑度**で分類する。
両者は直交しており、Layer 1/2 のコンポーネントは Review・Progress どちらにも配置しうる。

```
タブ（UIの分類）        Layer（計算の分類）
─────────────          ──────────────────
Review  = 期間振り返り    Layer 1 = 自動計算（KPI数値）
Progress = 累積データ     Layer 2 = パターン検出（チャート）
Insights = AI分析        Layer 3 = LLMインサイト
```

---

## 3層アーキテクチャ

```
Layer 1 (自動計算) — KPIメトリクス 8カード     ← 現在: Review タブ
  ↓ データを提供
Layer 2 (パターン検出) — チャート 4つ           ← 現在: Progress タブ
  ↓ 構造化データを提供
Layer 3 (LLMインサイト) — 自然言語の気づき生成  ← 現在: Insights タブ
```

※ 上記は現在の配置。タブとLayerは1:1ではなく、今後コンポーネントがタブ間を移動する可能性がある。

詳細: [Layer 1 Docs](?path=/docs/features-stats-layer1-docs--docs) | [Layer 2 Docs](?path=/docs/features-stats-layer2-docs--docs)

## コンポーネント構成

```
StatsPageContent                    タブ管理 + URL ルーティング
├── Review タブ
│   └── StatsView
│       └── StatsMetricsGrid       8カード KPI（2行×4列）
│           └── MetricCard         単一 KPI カード
│
├── Progress タブ
│   └── ProgressView
│       ├── YearlyHeatmap          年間ヒートマップ
│       ├── MonthlyTrendChart      月次トレンド
│       ├── DayOfWeekChart         曜日別分布
│       └── HourlyDistributionChart 時間帯分布
│
└── Insights タブ
    └── InsightsView               AI分析（プレースホルダー）
```

### ヘッダーコントロール

Review タブ選択時のみ表示:
- `DateNavigator` — 前後・今日ナビゲーション
- `StatsGranularitySelector` — 粒度切替（日/週/月/年）

## 状態管理

### useStatsFilterStore

| State | 型 | 説明 |
|-------|----|------|
| `granularity` | `'day' \| 'week' \| 'month' \| 'year'` | 表示粒度 |
| `currentDate` | `Date` | ナビゲーション基準日 |

タブの状態は URL がソース・オブ・トゥルース（`/stats/review`, `/stats/progress`, `/stats/insights`）。
`pushState` によるクライアントサイド遷移で高速切り替え。

## tRPC エンドポイント

### entries統計ルーター（`src/features/entry/server/statistics.ts`）

#### Layer 1: KPI メトリクス

| エンドポイント | Input | 説明 |
|-------------|-------|------|
| `getCumulativeTime` | dateRange | 期間内の記録時間合計 |
| `getStreak` | — | 連続アクティビティ日数 |
| `getPlanRate` | dateRange | 計画率 |
| `getEstimationAccuracy` | dateRange | タグ別見積もり精度 |
| `getEnergyMap` | dateRange | 時間帯×曜日エネルギー分布 |
| `getContextSwitches` | dateRange | タスク切替回数 |
| `getBlankRate` | dateRange + wake/sleep | 空白時間率 |

#### Layer 2: チャート

| エンドポイント | Input | 説明 |
|-------------|-------|------|
| `getDailyHours` | `{ year }` | ヒートマップ用日次集計 |
| `getMonthlyTrend` | `{ months? }` | 月次推移 |
| `getDayOfWeekDistribution` | dateRange? | 曜日別分布 |
| `getHourlyDistribution` | dateRange? | 時間帯分布 |

## データフロー

```
PostgreSQL RPC関数
  → entries統計ルーター
    → TanStack Query (並列取得・自動キャッシュ)
      → 純粋計算関数 (metrics.ts) or Recharts チャート
        → MetricCard / Chart コンポーネント
```
