# Search

グローバル検索。Cmd/Ctrl + K でどこからでもアクセス可能。

---

## 概要

| 項目 | 値 |
|------|-----|
| 起動 | `Cmd/Ctrl + K` |
| UI | 検索モーダル |
| 検索対象 | Tags、Blocks（エントリのメモ） |
| 状態管理 | React Context（`GlobalSearchProvider`） |
| データソース | クライアントサイド（`useTags`, `useEntries`） |

---

## コンポーネント構成

```
GlobalSearchProvider            Context プロバイダー + キーバインド
└── GlobalSearchModal           検索モーダル
    ├── Tags グループ           タグ一覧 / タグ名検索結果
    └── Blocks グループ         エントリ検索結果（入力時のみ）
```

---

## 動作

| 状態 | 表示内容 |
|------|---------|
| **空クエリ** | タグ一覧（カラードット + タグ名） |
| **入力あり** | タグ名マッチ + ブロック（description / タグ名マッチ） |

### 選択時の動作

| 対象 | 動作 |
|------|------|
| **Tag** | `showOnlyTag(tagId)` — そのタグだけフィルターON |
| **Block** | カレンダーでその日に遷移 + Inspector を開く |

---

## Hooks

| Hook | 用途 |
|------|------|
| `useGlobalSearch()` | Context hook（open, close, isOpen） |

---

## データフロー

```
Cmd/Ctrl + K → GlobalSearchModal 表示
       ↓
クエリ入力 → クライアント側フィルタリング
       ↓
├── Tags フィルタリング（タグ名）
└── Entries フィルタリング（description + タグ名）
       ↓
結果表示 → 選択
├── Tag → showOnlyTag でフィルター適用
└── Block → /calendar/day?date=... + Inspector 開く
```

---

## 連携

| 連携先 | 操作 |
|-------|------|
| `useEntryInspectorStore` | Block 選択時に Inspector を開く |
| `useCalendarFilterStore` | Tag 選択時に showOnlyTag でフィルター適用 |
