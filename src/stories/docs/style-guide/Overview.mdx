# Dayopt スタイルガイド

「装飾のない基本体験」を実現するためのUI設計リファレンス

---

## 設計原則

### 判断フローチャート

```
この要素は機能を伝えるために必要か？
├─ Yes → 実装する
│   └─ 最小限のスタイルで表現できるか？
│       ├─ Yes → そのまま実装
│       └─ No → 機能を分割して再検討
└─ No → 実装しない（装飾的要素は避ける）
```

### 採用するパターン

- ✅ 控えめなホバー状態（機能的フィードバック）
- ✅ セマンティックカラー（意味を伝える色使い）
- ✅ 一貫したスペーシング（8pxグリッド）
- ✅ シンプルなアイコン（機能を瞬時に伝える）

### 避けるパターン

- ❌ 派手なグラデーション背景（装飾的）
- ❌ 過剰なアニメーション（ノイズ）
- ❌ 影の多用（フラットで十分）
- ❌ イラスト・装飾画像（機能に寄与しない）

---

## GAFA-First 原則

UI/UXで迷ったら文脈に合うGoogle製品を開いて観察する。

- **UIパターン**: Material Design 3、Apple HIG準拠
- **コンポーネント**: shadcn/ui（Radix UI）を基本とする
- **カラー**: セマンティックトークンのみ使用
- **機能参考**: カレンダー → Google Calendar、時間記録 → Toggl
