# Typography

フォントサイズは Tailwind のデフォルトスケールをそのまま使用。業界標準で学習コストゼロ。

---

## 原則

- **サイズの差**で重要度を表現（大きい = 重要）
- **ウェイト**で強調（bold = 見出し、normal = 本文）
- **色**で階層（foreground = 主要、muted-foreground = 補助）
- フォント: Inter（英語）+ Noto Sans JP（日本語）、next/font で最適化済み

---

## フォントサイズスケール

| クラス | サイズ | 用途 | 備考 |
|--------|--------|------|------|
| `text-xs` | 12px | タイムスタンプ、注釈 | **よく使う** |
| `text-sm` | 14px | UIテキスト、ボタン、ラベル | **よく使う** |
| `text-base` | 16px | 本文（デフォルト） | **よく使う** |
| `text-lg` | 18px | 小見出し | |
| `text-xl` | 20px | カード見出し | |
| `text-2xl` | 24px | セクション見出し | **よく使う** |
| `text-3xl` | 30px | ページタイトル | |
| `text-4xl` | 36px | ヒーロー | |

---

## タイポグラフィ階層

UIで使う典型的なパターン。

| 用途 | クラス |
|------|--------|
| ページタイトル | `text-2xl font-bold` |
| サブテキスト | `text-sm text-muted-foreground` |
| セクション見出し | `text-lg font-bold` |
| カード見出し | `font-bold` |
| フォームラベル | `text-sm font-bold` |
| 補助テキスト | `text-xs text-muted-foreground` |

---

## フォントウェイト

2つだけ。シンプルに保つ。

| クラス | 用途 |
|--------|------|
| `font-bold` | 見出し、強調（ページタイトル、セクション見出し、ラベル、重要な情報） |
| `font-normal` | 本文テキスト（段落、説明文、UIテキスト全般） |

---

## 行間

| 用途 | クラス | 値 |
|------|--------|----|
| 見出し | `leading-tight` | 1.25 |
| 本文 | `leading-normal` | 1.5 |
| 長文 | `leading-relaxed` | 1.625 |

---

## Do's & Don'ts

### Tailwind デフォルトスケールを使用

```tsx
// ✅ Do
<h1 className="text-2xl font-bold">見出し</h1>
<p className="text-base">本文</p>
<span className="text-sm text-muted-foreground">補助</span>

// ❌ Don't — 任意値は禁止
<h1 className="text-[23px] font-bold">見出し</h1>
<p className="text-[15px]">本文</p>
```

理由: 業界標準のスケールで一貫性を保ち、デザインの統一感を維持。

### 階層的なサイズ変化

```tsx
// ✅ Do — 1段階ずつ下げる
<h1 className="text-xl font-bold">見出し1</h1>
<h2 className="text-lg font-bold">見出し2</h2>
<p className="text-base">本文</p>

// ❌ Don't — サイズが飛びすぎ
<h1 className="text-3xl font-bold">見出し1</h1>
<h2 className="text-sm font-bold">見出し2</h2>
<p className="text-xs">本文</p>
```

理由: 視覚的階層が崩れ、情報の重要度が伝わりにくくなる。

### ウェイトは2種類まで

```tsx
// ✅ Do
font-bold（見出し、強調）
font-normal（本文）

// ❌ Don't — font-black, font-extrabold, font-semibold, font-medium, font-light
```

理由: ウェイトの多用は視覚的ノイズになり、本当に強調したい部分が埋もれる。

### テキスト色で階層を表現

```tsx
// ✅ Do — セマンティックトークン
<p className="text-foreground">主要テキスト</p>
<p className="text-muted-foreground">補助テキスト</p>

// ❌ Don't — 直接色指定、透明度
<p className="text-gray-600">直接色指定</p>
<p className="text-foreground/50">透明度</p>
```

理由: セマンティックトークンでダークモード対応を自動化。
